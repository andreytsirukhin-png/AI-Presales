# MVP backlog

## Epic 1 — Document ingestion

- Upload PDF, DOCX and TXT.
- Validate size and extension.
- Extract text while retaining page and section metadata.
- Detect empty or scanned documents.
- Store document metadata.

## Epic 2 — Requirement extraction

- Define JSON schemas.
- Extract functional and non-functional requirements.
- Extract integrations, data, security and delivery constraints.
- Mark mandatory versus optional requirements.
- Attach evidence to every requirement.

## Epic 3 — Gap analysis

- Detect ambiguous wording.
- Detect missing versions, volumes, SLAs and API details.
- Detect contradictory dates and scope statements.
- Generate prioritized clarification questions.
- Link questions to requirements.

## Epic 4 — Risks and assumptions

- Generate delivery and technical risks.
- Score probability and impact.
- Generate mitigation actions.
- Generate explicit assumptions.
- Link risks and assumptions to source evidence.

## Epic 5 — Review and export

- Add accept, reject and edit actions.
- Store reviewer decisions.
- Export approved results to JSON.
- Add DOCX and XLSX exports after JSON is stable.

## Epic 6 — Evaluation

- Create five anonymized test RFPs.
- Produce manually labeled expected results.
- Measure extraction recall and precision.
- Measure evidence correctness.
- Track question usefulness using reviewer scoring.
